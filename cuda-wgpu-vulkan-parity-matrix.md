# CUDA/WGPU/Vulkan Parity Matrix

Date: 2026-06-11

This document is a current-state snapshot, not a completion claim. It turns the goal in [`goal.md`](../goal.md) into explicit parity rows tied to the current Candle checkout, with honest `wgpu`/`Vulkan` status against the CUDA backend.

## Legend

- `native`: GPU path exists and was exercised in this phase.
- `partial`: GPU path exists, but has dtype/layout/rank restrictions or missing test/model/profiling closure.
- `fallback`: code explicitly falls back to CPU on the current path.
- `unverified`: code exists but was not re-certified in this phase on RTX 3060.

## First ConvMixer Profiling Evidence

- Release single-case timings observed on the RTX 3060:
  - Vulkan before the allocator cleanup: `19.28s`
  - Vulkan after the allocator cleanup: `17.58s`
  - Vulkan after descriptor-set batching: `15.34s`
  - CUDA: `2.36s`
- Negative experiment that was measured and reverted:
  - dense rank-2 matvec row chunk `8 -> 16` regressed to `18.45s`
  - dense rank-2 matvec row chunk `8 -> 32` regressed to `22.29s`
- External direct-binary timing before the Vulkan cleanup:
  - Vulkan wall-clock: `16.12s`
  - CUDA wall-clock: `3.10s`
- External direct-binary timing after descriptor-set batching:
  - Vulkan wall-clock: `16.08s`
- `perf` is installed but unusable in this environment because `perf_event_paranoid=4`.
- `heaptrack` was used instead on the Vulkan ConvMixer case.
- First Vulkan `heaptrack` result:
  - total allocations: `5,642,954`
  - top host allocation root: `VulkanDevice::run_compute_specialized_with_options`
  - dominant temporary allocator: `ash::device::Device::allocate_descriptor_sets`
  - dominant model/backend path:
    - `VulkanStorage::conv2d`
    - `VulkanStorage::run_matmul_f32`
    - `VulkanStorage::run_rank2_matmul_f32_via_batched_matvec`
- First optimization pass:
  - replaced several per-dispatch `Vec` allocations in `candle-core/src/vulkan_backend.rs` with `SmallVec`
- Vulkan `heaptrack` after that pass:
  - total allocations: `3,611,920`
  - remaining hottest allocator: `ash::device::Device::allocate_descriptor_sets`
- Second optimization pass:
  - batch-allocate descriptor sets per layout inside an active Vulkan batch in `candle-core/src/vulkan_backend.rs`
- Vulkan `heaptrack` after descriptor-set batching:
  - total allocations: `3,104,162`
  - temporary allocations: `974,979`
  - `ash::device::Device::allocate_descriptor_sets` calls dropped from `676,910` to `105,396`
- Current profiling conclusion:
  - descriptor-set churn was a real bottleneck and is now materially reduced
  - the next highest-value Vulkan optimization target is the residual per-dispatch host overhead in `run_compute_with_shader`, plus a possible alternative dense matmul route for the ConvMixer conv2d path because larger row chunks regressed
